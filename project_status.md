# 서대전여고 통합 플랫폼 서버 (SSO) 프로젝트 명세서

## 1. 프로젝트 개요
서대전여고의 모든 웹앱(스마트 시간표, 학생용 AI, 급식 관리 등)을 중앙에서 관리하기 위한 **'통합 인증 및 포털 서버(SSO)'** 구축 프로젝트.
기존 개별 웹앱들이 각자 로그인 로직을 처리하던 방식에서 벗어나, 본 플랫폼 서버가 회원 데이터(구글 시트)를 독점 관리하고 인증 토큰(JWT)을 발급하는 마이크로서비스 아키텍처(MSA)를 도입한다.

## 2. 데이터베이스 (구글 스프레드시트 구조)
사용자 관리는 직관성과 관리 편의성을 위해 '구글 스프레드시트'를 마스터 DB로 사용한다. (A~I열)

| 열(Column) | 항목명 | 설명 및 사용법 |
|---|---|---|
| **A열** | 구글 계정 | 구글 간편 로그인 시 매칭되는 계정 이메일 |
| **B열** | 일반 ID | 방화벽 환경에서 사용하는 일반 로그인 아이디 (주로 학번 또는 교직원 번호) |
| **C열** | 비밀번호 | 일반 로그인용 암호화된 비밀번호 (초기엔 평문 부여 후 강제 변경 시 bcrypt 해싱) |
| **D열** | 이름 | 실제 사용자 이름 |
| **E열** | 고유식별자 | 학번(예: 10101) 또는 교사ID(예: T-038). 권한 분기 및 앱 간 연동의 핵심 키 |
| **F열** | 권한 | 관리자 / 업무담당자 / 교사 / 학생 / 학부모 / 졸업생 등 |
| **G열** | 상태 | Active(정상) / Inactive(정지) / Pending(승인 대기) |
| **H열** | 가입일시 | 계정이 최초 생성된 날짜 및 시간 |
| **I열** | 비번변경필요 | TRUE일 경우 일반 로그인 시 강제로 비밀번호 변경(암호화) 유도 |

## 3. 핵심 아키텍처 (SSO 동작 원리)
1. **플랫폼 로그인**: 사용자가 포털에 접속하여 구글 계정 또는 일반 아이디/비번으로 로그인.
2. **검증 및 토큰 발급**: 플랫폼 서버가 구글 시트를 조회하여 권한을 확인하고, 서명된 JWT 토큰 발급.
3. **토큰 전달 (슈퍼 앱 방식)**: 사용자가 플랫폼 메뉴에서 '스마트 시간표'를 클릭하면 `time.sdjgh.kr?token=xxx...` 형태로 토큰을 전달.
4. **서브 앱 프리패스**: 시간표 서버는 자체 로그인 검사 없이, 플랫폼 서버와 공유하는 `JWT_SECRET` 키를 통해 토큰의 위조 여부만 판별한 뒤 즉시 접속 허용.

## 4. 첫 번째 개발 목표 (마일스톤)
1. **플랫폼 백엔드 (Node.js/Express) 구축**:
   - `bcrypt` 라이브러리를 활용한 비밀번호 해싱 시스템 도입.
   - 구글 Oauth2 및 일반 로그인 API 통합 라우트 설계.
2. **플랫폼 프론트엔드 (React) 구축**:
   - 서대전여고 슈퍼 앱 포털 메인 화면 구성.
   - '최초 로그인 시 비밀번호 강제 변경' UI 구현.
3. **타 웹앱(스마트 시간표) 연동 테스트**:
   - 플랫폼에서 발급한 토큰을 스마트 시간표가 검증하고 받아들이도록 연동.
